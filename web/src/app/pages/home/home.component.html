Cache Proxy

可用代理实例

展开任意实例即可查看它的实际入口地址和对应客户端配置片段，直接复制即可接入。

过期时间语义

ExpireAfter（过期时间）

  • 不配置：使用默认值 720h（30天）
  • 720h：720小时后过期
  • never 或 0：永不过期
  • -1：不允许（负数报错）

FreshFor（新鲜度窗口）

  • 不配置：不进行新鲜度检查，内容始终视为过期
  • 1h：1小时内视为新鲜
  • forever 或 0：永远新鲜

各模块配置说明

npm 包代理

  • expire_after: 缓存过期时间，默认 720h
  • fresh_for: 新鲜度窗口，forever=永远新鲜
  • policy: bypass/immutable/revalidate

PyPI 包代理

  • expire_after: 缓存过期时间，默认 720h
  • simple_fresh_for: 简单索引新鲜度
  • 支持 pip、pip.conf 两种配置片段

Go 模块代理

  • GOPROXY: 设置代理地址即可
  • GONOSUMDB: 跳过校验的和
  • 支持私有 GOPROXY 场景

Maven/Gradle 代理

  • expire_after: 缓存过期时间，默认 720h
  • metadata_fresh_for: 元数据新鲜度
  • snapshot_fresh_for: SNAPSHOT 版本新鲜度
  • 支持 pom.xml 和 Gradle 两种配置

Cargo 包代理

  • index_fresh_for: 索引新鲜度
  • sparse registry 协议支持
  • 自动配置 protocol 和 index

OCI 容器镜像代理

  • expire_after: 镜像缓存过期时间
  • fresh_for: 清单新鲜度
  • 支持 docker pull 直接配置

主动清理配置

  • enabled: 是否启用主动清理，默认 false
  • interval: 清理扫描间隔，默认 6h
  • dry_run: true=只记录不删除
  • batch_size: 每批删除对象数，默认 500
  • workers: 并行清理工作数，默认 CPU 数

GC 配置

  • blob: Blob GC 间隔，默认 24h，0=使用默认值
